Other ideas for upcoming releases:

*** Personalization & UX
- Diary-level theming and colors so entries are visually distinguishable in the All Diaries view
- Custom tag colors and categories for visual organization of tags
- Keyboard shortcuts for common actions (Ctrl+N new entry, Ctrl+/ focus search, Esc close modals, arrow keys for pagination)
- Calendar heatmap view (GitHub-style contribution grid) showing journaling activity by day in stats
- Recurring journaling reminders and writing prompts via app notifications at a user-configured time
- Drag-and-drop reordering for entries on the same day
- Voice personalization for TTS

*** Accessibility
- Full accessibility audit and remediation (aria-labels, screen reader support, skip-to-content links, focus management)
- High contrast mode and adjustable font sizes
- Keyboard-only navigation support for all interactive elements

*** Infrastructure & Reliability
- Health check endpoint (Kubernetes probes reference /api/health but none exists in the codebase)
- Database migration files instead of relying on TypeORM synchronize (add a migrations/ folder with versioned migration scripts)
- Complete the email verification flow (User entity has emailVerified field but no verification endpoint or email is implemented)
- Database connection pooling configuration in TypeORM data source
- Structured JSON logging for observability and request tracing
- Content Security Policy hardening (replace unsafe-inline with nonces)
- CI pipeline for automated dependency vulnerability scanning (npm audit, Dependabot/Renovate)
- Blue-green or canary deployment strategy for zero-downtime releases
- Database read replicas for scaling read-heavy queries (stats, feed, search)
- Automated database backups with point-in-time recovery
- Distributed rate limiting with Redis for multi-replica deployments
