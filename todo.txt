*** Import / Export
- Import from a file will use the configured default visibility of the diary
- The filename of the export is the name of the diary + the date of the export
- Ability to choose whether to export the visibility field or not, by default set to false
- Export to a file needs to add a visibility field like --[public/private] just after the tags.
- Manage the behavior of the import/export when using All Diaries
- Supporting TXT, JSON, and MD format import export

- Content Security Policy hardening (replace unsafe-inline with nonces)
- CI pipeline for automated dependency vulnerability scanning (npm audit, Dependabot/Renovate)

- Automated database backups with point-in-time recovery
